# 车辆本体（硬件描述 / URDF）

> 逐一核对自 `small_brain/urdf/mini_mec_six_arm_sim.urdf`（仿真用 URDF，SolidWorks 导出 + 手工增补传感器/Gazebo 插件）。
> 本文档汇总当前仿真 URDF 的坐标系、关节、传感器和 Gazebo 插件配置。

机器人 = 麦克纳姆轮底盘 + 六自由度机械臂 + 二指夹爪 + RGB-D 相机。

---

## 1. 坐标系与尺寸

`base_link` 原点在**地面**（z=0 = 地面）。

| 项 | 值 | 来源 |
|----|----|------|
| 底盘长 × 宽 | 285.4 × 230 mm | footprint 半长 0.143、半宽 0.115 |
| 轮径 | 75 mm | 轮 z 偏移 0.038 |
| 前轮中心 | x=+0.0950, y=±0.0767, z=+0.0380 | joint origin |
| 后轮中心 | x=−0.0780, y=±0.0767, z=+0.0380 | joint origin |
| 机械臂基座 joint1 | x=+0.0545, y=+0.0007, **z=0.156** | joint1 origin |
| 相机 camera_link | x=−0.1143, y=+0.0008, z=0.556, pitch=0.06 | camera_joint |

> base_link **+X = 车头**（机械臂 joint1 在 +X 侧，相机镜头也朝 +X）；−X = 车尾。

## 4. Gazebo 插件（在 URDF `<gazebo>` 内）

| 插件 | 文件 | 作用 |
|------|------|------|
| ros_control | `libgazebo_ros_control.so` | 驱动 arm/hand 控制器 |
| 麦轮底盘 | `libgazebo_ros_planar_move.so` | 订阅 `/cmd_vel`，发 `odom`（frame=odom，20Hz）+ `odom→base_link` TF |
| 真值里程计 | `libgazebo_ros_p3d.so` | 发 `/ground_truth/odom`（frame=**world**，50Hz，零噪声，publishTf=0）；RTAB-Map 用它 |
| **抓取稳定** | `libgazebo_grasp_fix.so` | palm=link5，gripper_link=link6–11；grip_count_threshold=2，release_tolerance=0.005 |

> 抓取流程同时包含 `gazebo_grasp_fix` 和 `block_follower_plugin`；搬运由 `block_follower_plugin` 通过 `SetWorldPose` 跟随 `link5` 完成，详见《05_其他》。

### 轮/夹爪接触参数

- **车轮**：`mu1=mu2=0.0, kp=1e6, kd=100`（消除与 planar_move 的摩擦冲突）。
- **夹爪 link6–11**：`mu1=mu2=50, kp=50000, kd=10, minDepth=0.001, maxVel=0`；joint6–11 加 `implicitSpringDamper=true`（解决低惯量 ODE 不稳定）。

### Transmission（关节硬件接口）

**主臂与夹爪关节使用 `PositionJointInterface`，四轮使用 `VelocityJointInterface`。**
joint6–11 与 `sim_controllers.yaml` 中的 `position_controllers/JointTrajectoryController` 对应。

## 5. 相机（三个）

### 5.1 Orbbec Astra S（主相机，建图/导航/感知）

| 项 | RGB | Depth |
|----|-----|-------|
| 插件 | `libgazebo_ros_camera.so` | `libgazebo_ros_openni_kinect.so` |
| 分辨率 | **640×480** | **640×480** |
| HFOV | 1.1013 rad (63.1°) | 1.0194 rad (58.4°) |
| 帧率 | 30 fps | 30 fps |
| 深度范围 | — | clip 0.3–8.0 m，点云 cutoff 0.5–8.0 m |
| frameName | `camera_depth_optical_frame` | `camera_depth_optical_frame` |

> 两个 sensor 统一用光学坐标系 `camera_depth_optical_frame`（标准 ROS：z 前、x 右、y 下），避免 RGB/Depth 投影偏差。
> Topic：`/camera/rgb/image_raw`、`/camera/rgb/camera_info`、`/camera/depth/image_raw`、`/camera/depth/camera_info`、`/camera/depth/points`。
> `camera_joint` 与 `chase_cam_joint`、`judge_camera_joint` 均加 `preserveFixedJoint=true`，防止 Gazebo 把相机并进 base_link。

### 5.2 chase_cam（观察相机）

- 位置：base_link 后退 0.5m、升高 1.1m，俯角 **pitch=0.70 rad ≈ 40°**。
- 分辨率 **2560×1440** @ 5Hz，HFOV 1.4 rad。
- Topic：`/chase_cam/image_raw`。由 `big_brain/vlm_image_writer.py` 写成 `image/image.png` 供 VLM。

### 5.3 judge_camera（E2 VLM judge 用）

- 位置：车左侧 0.75m、高 0.55m、略前 0.25m，向右看向车身（yaw=−π/2），俯角 0.35 rad。
- 分辨率 1280×720 @ 5Hz，HFOV 1.4 rad。
- Topic：`/judge_camera/rgb/image_raw`。

## 6. TF 树

```
map →(static identity)→ world →(static identity)→ odom →(planar_move 20Hz)→ base_link
  └→ link1…link5（robot_state_publisher 50Hz）
  └→ camera_link → camera_depth_optical_frame
  └→ chase_cam_link / judge_camera_link → *_optical_frame
```

定位由 Gazebo 真值负责，RTAB-Map `publish_tf=false`（不进 TF 树）。详见《02_RTAB-Map》。
