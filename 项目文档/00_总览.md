## 系统架构

```
用户自然语言指令
        │
        ▼  big_brain/（纯 Python，低频语义推理）
┌──────────────────────────────────────────────┐
│ RAG 检索(all-mpnet, ≥0.5) → Planner(DeepSeek)  │
│ 生成 Python 脚本 → exec                         │
│ 每原语后 JudgeLLM：规则 + VLM(Gemini) 判定      │
│ 失败 → replan(per-block 配额) 生成恢复代码      │
└───────────────────┬──────────────────────────┘
                    │ robot_api 原语 / ros_bridge
                    ▼  small_brain（small_brain_sim，本地实时）
┌──────────────────────────────────────────────┐
│ RTAB-Map 视觉 SLAM（RGB-D → /rtabmap/grid_map）│
│ move_base + GlobalPlanner + TEB 导航           │
│ arm_task_server（/arm/pick·put·drop·home）     │
│ block_follower_plugin（1000Hz SetWorldPose 搬运）│
└──────────────────────────────────────────────┘
        ▲  Gazebo 11（小屋场景 + URDF + 真值里程计）
```

行动原语 ↔ JudgeLLM action_id 映射：1=move_to_xy, 2=move_to_obj_by_offset,
3=pick_up_xy, 4=pick_up_obj, 5=put_down_xy, 6=put_down_obj_by_offset（详见《04》）。

## 启动流程（TEB 主流程）

```bash
# 1 Gazebo 仿真（含小屋 + 机械臂初始化 + chase/judge 相机）
./scripts/run_gazebo.sh
# 2 RTAB-Map（首次 ./run_rtab.sh map 建图，之后 ./run_rtab.sh 定位）
./scripts/run_rtab.sh
# 3 TEB + move_base
./scripts/run_teb_compare.sh
# 4 抽取场景对象坐标（一次性）
python3 scripts/get_scene.py --deploy
# 5 机械臂服务
python3 small_brain/scripts/arm_task_server.py
# 6 VLM 图像桥接（大脑判定用）
python3 big_brain/vlm_image_writer.py &
# 7 大脑（自然语言任务）  /  或小脑独立任务 scripts/pick_deliver_teb.py
python3 big_brain/big_brain.py
```

依赖顺序：2 等 1 的 `/camera/depth/image_raw` → 3 等 2 的 `/rtabmap/grid_map` → 7 等 3、5 就绪。
环境与编译见 [../ENVIRONMENT.md](../ENVIRONMENT.md)。
