# 实验说明

本目录保存系统实验评估脚本，只保留复现实验所需代码，不包含运行后生成的 `outputs/`、CSV、JSON、jsonl、截图或临时证据文件。这些结果文件会在本地生成，并由 `.gitignore` 忽略。

运行实验前建议先设置项目根目录：

```bash
export PROJECT_ROOT="$(pwd)"
```

如果当前终端位于 `experiments/` 目录内，则使用：

```bash
export PROJECT_ROOT="$(cd .. && pwd)"
```

## 任务分组

| 分组 | 内容 | 主要考察能力 | 说明 |
|------|------|--------------|------|
| [N1-N4](N1-N4/README.md) | 导航任务 | 坐标导航、目标物接近、环绕路径、往返路径 | 不涉及机械臂与 VLM |
| [G1-G4](G1-G4/README.md) | 抓取与放置任务 | 原地抓取、导航后抓取、桌面放置、两物体之间放置 | 涉及机械臂、夹爪、放置判定 |
| [C1-C4](C1-C4/README.md) | 复合任务脚本 | 长程任务分解、多物体循环、巡逻中断与恢复 | 主复合评估聚焦 C3/C4，C1/C2 作为局部能力脚本保留 |

## 实验版本区别

| 版本 | 方法 | 是否调用 LLM | 是否执行 Gazebo | 是否调用 VLM | 是否允许重规划 | 用途 |
|------|------|--------------|-----------------|--------------|----------------|------|
| E1 | 大脑主流程 | 是 | 是 | G/C 会调用，N 不需要 | `adjust_policy=2` | 主实验，评估完整系统成功率 |
| E2 | 闭环消融 | 是 | 是 | G/C 会调用，N 不需要 | `adjust_policy=0/1` | 比较无重规划与一次重规划的差异 |
| E3 | 静态规划 | 是 | 否 | 否 | 否 | 只看模型生成代码是否语义正确、分解正确、可执行 |
| E4 | 人工 API 基线 | 否 | 是 | 否 | 否 | 作为小脑能力和人工流程上限参考 |

说明：

- N 组是纯导航能力测试，成功与否主要由里程计误差和路径完成情况判定。
- G 组是抓取/放置能力测试，最终结果以 Gazebo 中物体位置、抓取高度和放置误差判定。
- C 组是复合任务测试，除最终物体状态外，还记录执行阶段、重规划、VLM 判定和附录证据；主统计使用 C3/C4 no-helper，C1/C2 不重复纳入主复合结果。
- E1/E2 中的 `adjust_policy` 表示每个任务块允许的恢复次数；数值越高，系统越可能通过自验证与重规划纠偏，但调用成本和执行时间也会上升。
- E3 不启动 Gazebo，适合低成本比较不同模型、温度、RAG 或提示词版本的规划质量。
- E4 不经过 LLM/VLM，使用人工写定的 API 序列，主要用来说明机器人小脑执行链本身是否具备完成任务的能力。

## 版本开关

| 开关 | 适用范围 | 含义 |
|------|----------|------|
| `--models flash pro` | E3 | 比较不同规划模型生成代码的能力 |
| `--temperatures 0.0 0.4 0.8` | E3 | 比较不同采样温度下的稳定性 |
| `--no-rag` | G 组 E3 | 关闭历史任务检索，只使用基础提示词 |
| `--no-helper-prompt` | C 组 E1/E2/E3 | 移除 C1-C4 专用 helper 提示，要求模型从基础原语组合任务 |
| `--c1-block-variant` | C 组 E1/E2 | 将 C1 的 Coke 罐版本切换为 red_block 版本 |
| `--start-writer` | G/C 组 E1/E2 | 启动 VLM 图像写入进程，持续保存 judge_camera 图像 |
| `--continue-on-error` | pipeline 脚本 | 单次任务失败后继续跑后续任务，便于批量收集失败样本 |

## 运行前置条件

执行 E1/E2/E4 前通常需要先启动：

```bash
./scripts/run_gazebo.sh
./scripts/run_rtab.sh
./scripts/run_teb_compare.sh
python3 scripts/get_scene.py --deploy
python3 small_brain/scripts/arm_task_server.py
```

E3 只请求模型生成代码，不需要 Gazebo、RTAB-Map、TEB 或机械臂服务。

## 输出文件

各组脚本会在对应目录生成 `outputs/` 或 CSV 文件。临时输出、截图和事件流水不放入代码目录；如需整理结果表格，可从 CSV 统计成功率、调用次数、执行时间、误差和失败原因。
